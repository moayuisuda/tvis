# Tvis Spec Guide

📊 Terminal chart renderer with G2 spec support. This doc guide you how to generate a tvis spec.

<Basic Spec Example>
{
  "type": "interval",
  "data": [
    { "category": "A", "value": 10, "type": "X" },
    { "category": "A", "value": 8, "type": "Y" },
    { "category": "B", "value": 12.23, "type": "X" },
    { "category": "B", "value": 9, "type": "Y" },
    { "category": "C", "value": 4, "type": "X" },
    { "category": "C", "value": 2, "type": "Y" }
  ],
  "axis": {
    "y": {
      "title": "Sales",
      "label": {
        "formatter": "$.1~f"
      }
    },
    "x": {
      "title": "Category"
    }
  },
  "label": {
    "formatter": ".1~f"
  },
  "encode": {
    "x": "category",
    "y": "value",
    "color": "value"
  }
}
</Basic Spec Example>

<Spec Options>
- type: "interval" | "line" | "point"
- data: Array
- encode:
  - x: field name
  - y: field name
  - color?: field name
- transform?:
  - { type: "stackY" }
  - { type: "dodgeX" }
- coordinate?:
  - { type: "transpose" }
- axis?:
  - { x?: AxisSpec, y?: AxisSpec }
  - AxisSpec: { title?: string, label?: boolean | { formatter?: string | ((v: string | number) => string) } }
- label?: boolean | { formatter?: string | ((v: string | number) => string) }
- legend?: { color?: false }
- title?: string
- colors?: string[]
- mode?: "ascii" | "color"
</Spec Options>

<More Examples>
# Transpose (act like bar chart)
{
  "type": "interval",
  "data": [
    { "month": "Jan", "sales": 100 },
    { "month": "Feb", "sales": 120 },
    { "month": "Mar", "sales": 150 }
  ],
  "encode": {
    "x": "month",
    "y": "sales"
  },
  "coordinate": {
    "type": "transpose"
  }
}

# Linear Scale ( when you need a linear gradient in numerical values )
{
  "type": "point",
  "colors": ["#28a745", "#dc3545"],
  "data": [
    { "height": 85, "weight": 44 },
    { "height": 172, "weight": 63 },
    { "height": 180, "weight": 150 },
    { "height": 180, "weight": 75 },
    { "height": 188, "weight": 86 },
    { "height": 20, "weight": 12 }
  ],
  "encode": {
    "x": "height",
    "y": "weight",
    "color": "weight"
  }
}
</Examples>

## Tips
0. format string is completely consistent with d3-format.
1. for line/point type, do not append `label` attr by default.
2. `transpose` only work with interval.
3. if label is append, use { format: .2~f } by default.

## output
1. only output pure json string.
